# Глобальный аудит проекта и план задач (Обновлено)

## 1. Результаты аудита текущей кодовой базы
Я провел подробный аудит директорий `backend/` и `bot/`. Выяснилось, что проект реализован на **очень высоком техническом уровне**, намного превосходящем классический студенческий MVP. Многие задачи из изначального roadmap (`AI_INSTRUCTIONS.md`) *уже реализованы* и работают блестяще.

### ✅ Что уже реализовано на "отлично":
1. **Геймификация (Gamification):** База данных (`models/user.py`) уже содержит поля `xp`, `level`, `streak`. Логика обновления реализована: начисляется +15 XP за ревью текста и +30 XP за аудио-ревью. Работает расчет уровней и стриков по дням.
2. **Векторный поиск (pgvector & ML):** В `api/recommendations.py` и `services/ml.py` полностью реализован сложный алгоритм персонализации! Он учитывает настроение пользователя, время суток, сезон и сгенерированный контекст, используя векторное представление через эмбеддинги (`paraphrase-multilingual-MiniLM-L12-v2`).
3. **Парсинг в реальном времени (Web Scraping):** `services/parser.py` умеет делать запросы к Yandex Search API v2 для поиска стихов прямо из интернета в момент запроса пользователя, если нужного стихотворения нет в базе.
4. **Оценка голоса (Voice Evaluation):** Оффлайн распознавание Vosk с конвертацией `ogg` в сырой `pcm` через ffmpeg. Отлично написан алгоритм сравнения текстов через `difflib.SequenceMatcher` со скользящим окном и стоп-словами.
5. **Миграции БД:** Инициализирован Alembic (папка `alembic` и `alembic.ini` присутствуют).

### ❌ Что отсутствует (Технический долг и слабые места):
1. **Автоматизированное тестирование (Критично для Assignments 4 & 5):** В бэкенде **полностью отсутствует папка `tests/`**. По требованиям курса у вас должно быть минимум 5 Unit и 5 Integration тестов. Сейчас их ноль.
2. **Таблица лидеров (Leaderboard):** Опыт (XP) копится, уровни растут, но пользователи не могут соревноваться друг с другом, так как нет рейтинга.
3. **Режим Stanza-by-stanza:** Из изначального плана не реализован построчный режим заучивания. Сейчас бот всегда вынужден прослушивать стихотворение целиком.
4. **Отсутствие CI/CD пайплайнов:** Нет `.gitlab-ci.yml` или `.github/workflows` для автоматической проверки кода.

## 2. Задачи по дальнейшей доработке (Новый Roadmap)
Для того чтобы получить идеальный балл за проект (Assignments 4-7) и улучшить продукт, я предлагаю следующий план работы:

**Фаза 1: Устранение долгов по тестированию (Приоритет: Высокий)**
*   Создать пакет `backend/tests/unit/` и `backend/tests/integration/`.
*   Написать минимум 5 юнит-тестов (на алгоритм SM-2 в `calculate_sm2` и на точность сравнения строк в `compare_texts`).
*   Написать 5 интеграционных тестов для FastAPI (проверка эндпоинтов рекомендаций и создания юзера через тестовую БД).
*   Добавить файл настройки CI Pipeline (например, запуск `pytest` и `ruff`).

**Фаза 2: Доработка геймификации (Leaderboard)**
*   Создать новый эндпоинт `GET /users/leaderboard` в FastAPI, который будет отдавать ТОП-10 пользователей, отсортированных по уровню и XP.
*   Добавить в Telegram-боте команду `/leaderboard` с красивым форматированием (с медалями 🥇🥈🥉).

**Фаза 3: Отладка Push-уведомлений (Scheduler)**
*   В боте есть файл `scheduler.py` и роут `/due/all` в бэкенде. Нужно проаудировать и убедиться, что бот реально стабильно "просыпается" каждый день и отправляет пользователям сообщения: *"Привет! У тебя 3 стихотворения на повторение"*, опираясь на `notification_time`.

**Фаза 4: Построчный режим изучения (Stanza-by-stanza)**
*   Сложная фича: добавить возможность разбивать длинные стихи на строфы (stanzas) и требовать от пользователя голосовые сообщения поочередно для каждой строфы.
